# 要件定義書

## 1. 目的
- 日本史をテーマにしたリファレンスサイトを構築する。
- 利用者が「時代」「人物」「タグ」から横断的に情報へ到達できるようにする。
- GitHub Pages で公開し、無料かつ低運用コストで継続運用可能にする。

## 2. スコープ
- 対象: 静的コンテンツ中心の閲覧サイト
- 非対象: ユーザー認証、管理画面、サーバーサイド API

## 3. 機能要件
- ヘッダー表示（サイト名、説明）
- サイドバー表示（時代一覧、タグ一覧）
- メイン表示（記事一覧、記事詳細）
- Markdown + frontmatter による記事管理
- クライアントサイド全文検索（Fuse.js）
- 時代・タグによる絞り込み表示
- GitHub Pages へのデプロイ

## 4. 非機能要件
- 静的ホスティングで動作すること
- ページ遷移は SPA で快適に動作すること
- モバイル幅でも崩れないレスポンシブレイアウト
- 記事追加時にコード改修を最小化できる構造

## 5. 前提技術
- フロントエンド: React + Vite + React Router
- UI: CSS Modules
- コンテンツ: `content/*.md` + frontmatter
- 検索: Fuse.js
- デプロイ: GitHub Pages (GitHub Actions)
